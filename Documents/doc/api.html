arena.models

Database Models for Prompt Arena

This module defines the SQLAlchemy data models used in the Prompt Arena application. It includes: - User: Handles authentication, profile data, OTP login, and guest/full account status. - PromptSubmission: Stores user-submitted prompts and AI-generated evaluations.

The models use Flask-SQLAlchemy and integrate with Flask-Login via UserMixin.

arena.routes

Application Routes Module (Blueprint)

This module defines all web routes for the Prompt Arena application using a Flask Blueprint. It handles: - User authentication via email/OTP and Google OAuth - Prompt submission and evaluation flow - Navigation to core pages (welcome, leaderboard, payment)

All routes require login unless explicitly public (e.g., login, verify-otp). Uses flask_login for session management and SMTP for OTP delivery.

arena.routes.login()

Handle user login via email and OTP.

On GET: Display the login form. On POST: - Validate required fields (first name, last name, email). - Create a new user if email doesn’t exist. - Generate a TOTP-based OTP and store it in the database. - Send OTP via email using send_otp_email.

Returns:

Rendered login template or redirect to OTP verification.

Return type:

str or werkzeug.wrappers.response.Response

Send a one-time password (OTP) to the specified email address.

Uses Gmail SMTP (or any SMTP server defined in Config) to deliver a plain+HTML email containing the 6-digit OTP. Used during login/signup for secure access.

arena.config

Application Configuration Module

This module defines environment-specific configuration settings for the Prompt Arena application using a class-based approach. It supports different modes: development, production, and testing.

The configuration is loaded from environment variables (via .env) with sensible defaults for local development.

Key Features: - Centralized settings for Flask, SQLAlchemy, email, OAuth, and OpenAI - Support for multiple environments - Secure defaults (e.g., TLS, CSRF) - Dynamic upload path creation

This file is typically used with:

app.config.from_object(config.config_by_name[env])

Where env is ‘development’, ‘production’, etc.
